# Terminology

* `container` The completed container. Think of it as a running VM.
* `image` The built image that the container will be running.
* `build steps` The steps used to build an image
* `Dockerfile` A recipy based describing the build steps

</Layout>

---

<Layout>

## The "Ikea" of Docker

1. `Dockerfile`: It all starts with such a file
2. Someone(thing) will build an `image` based on the `Dockerfile`, Docker executing the `build steps` described
3. Someone will use the `image` to run a `container`

</Layout>

<Layout>

## Some more concepts

1. Network - A virtual network created between containers
2. Volume - A shared (if specified) drive/volume/directory
