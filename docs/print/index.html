    - Introduction

    Docker is a set of platform as a service (PaaS) products that use OS-level virtualization to deliver software in packages called containers. Containers are isolated from one another and bundle their own software, libraries and configuration files; they can communicate with each other through well defined channels. All containers are run by a single operating-system kernel and are thus more lightweight than virtual machines.

    Conclusion

    Docker is a virtualization solution, more lightweight, therefore easier to use in development and production, as well as to scale, than existing VM solutions. Just by using docker you can create a self-contained ecosystem that can scale easily, and be deployed prebuilt / pre-configured, with little or no overhead.

    Intro to Docker

    What have we learned?

    • docker is the main command used.
    • To execute something, we need to use the docker run command
    • To build something, we need to use the docker build command
    • To see what docker processes (containers) are running use the docker ps command
    • To kill a process, we need to use the docker kill command
    • To execute code on an already existing container, we need to use the docker exec command
    • All of these commands have a lot of arguments

    Intro to Docker

    - Docker - Run

    Arguments:

    • --name: Will give the container a name
    • -i: Will run in interactive mode
    • -t: Will run in tty mode (will allow terminal use while interactive)
    • -it: A combination of the above. In translation: open a terminal connection
    • -d: Will run in daemon mode (background process)
    • --rm: Will remove the container on exit
    • <command>: Will override the default command
